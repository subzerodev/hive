Open Redirect

Open redirect vulnerabilities allow attackers to redirect users to malicious websites using trusted domain links.

Vulnerable Test Cases

  • Basic Redirect - Location header redirect
    ?url=https://evil.com
  • Meta Refresh - HTML meta tag redirect
    ?url=https://evil.com
  • JavaScript Redirect - window.location redirect
    ?url=https://evil.com
  • Login Parameter - Post-login redirect
    ?next=https://evil.com/phishing
  • Double Encode Bypass - URL encoding bypass
    ?url=%2568ttps://evil.com

False Positives (Safe)

  • Whitelist - Only allows specific paths
  • Domain Check - Only allows same domain
