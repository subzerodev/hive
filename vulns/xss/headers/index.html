XSS via HTTP Headers

XSS vulnerabilities where input comes from HTTP request headers instead of URL/form parameters.

Vulnerable Test Cases

  • Referer Header - Referer reflected without escaping
    curl -H "Referer: <script>alert(1)</script>" URL
  • User-Agent Header - User-Agent reflected without escaping
    curl -A "<script>alert(1)</script>" URL
  • Cookie Value - Cookie value reflected without escaping
    Set XSS cookie
  • X-Forwarded-For Header - XFF reflected without escaping
    curl -H "X-Forwarded-For: <script>alert(1)</script>" URL
  • Accept-Language Header - Accept-Language reflected without escaping
    curl -H "Accept-Language: <script>alert(1)</script>" URL

False Positives (Safe)

  • Properly Escaped - All headers HTML-escaped
