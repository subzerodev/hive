JWT Vulnerabilities

JSON Web Token (JWT) implementation vulnerabilities.

Vulnerable Test Cases

  • JWT Login - Basic JWT authentication flow
    Credentials: admin / password
  • None Algorithm - Accepts alg:"none" with no signature
    Forge tokens without secret
  • Weak Secret - Secret is "secret" (brute-forceable)
    Crack with jwt-cracker or hashcat
  • No Expiry Check - Expired tokens still accepted
    Stolen tokens work forever
  • URL Parameter - JWT in URL (leaks via referrer)
    Token visible in logs, history

False Positives (Safe)

  • Properly Validated - Algorithm whitelist, expiry check
