Server-Side Template Injection (SSTI)

SSTI occurs when user input is embedded into server-side templates without proper sanitization, allowing attackers to inject template directives.

Vulnerable Test Cases

  • Go text/template - User input in template string
    Try: {{printf "%s" "injected"}}
  • Go html/template - HTML auto-escaped but template logic works
    Try: {{.}}
  • Jinja2 Style - Simulated Jinja2/Python template
    Try: {{7*7}} or {{7*'7'}}
  • ERB Style - Simulated Ruby ERB template
    Try: <%=7*7%>

False Positives (Safe)

  • Properly Escaped - Input passed as data, not template code
